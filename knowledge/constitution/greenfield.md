# Greenfield Constitution

Rules for new feature and project work.

## Start with purpose

- Define who the feature serves and what problem it solves.
- Constitution-first: establish quality, security, testing, and performance
  standards before writing code.

## Delivery sequence

1. Constitution (principles and guardrails)
2. Specification (scope, acceptance criteria, NFRs)
3. Clarification (resolve ambiguities)
4. Architecture review (against principles)
5. Implementation plan (phases, risks, rollback)
6. Task breakdown (traceable, testable units)
7. Implementation (code + tests in parallel)
8. Quality validation (automated gates)

## Rules

- No implementation tasks before a plan exists.
- Acceptance criteria must be testable.
- Every feature needs at minimum: spec, plan, tasks, quality gate.
