# GitHub Copilot — Sync Map

Maps canonical `.apm/` capabilities to their Copilot provider-layer files
in `providers/github-copilot/`. When a canonical file is added or changed,
the corresponding provider file must be created or updated.

> **Architecture**: `.apm/` is the single canonical source of truth.
> `providers/github-copilot/{agents,prompts,instructions}` is the provider
> adapter layer (Layer 2). `.github/` is a runtime projection generated by
> `.apm/scripts/powershell/project-copilot.ps1` (Layer 3).

## Sync rules

1. **Adding an agent**: Create `.apm/agents/<name>.md` (canonical) → create `providers/github-copilot/agents/<name>.agent.md` (provider, with Copilot frontmatter) → run `project-copilot.ps1` → verify parity warning is gone
2. **Adding a prompt**: Create `.apm/prompts/<name>.md` → create `providers/github-copilot/prompts/<name>.prompt.md` → run `project-copilot.ps1`
3. **Adding a workflow**: Create `.apm/workflows/<name>.yml` → create `providers/github-copilot/prompts/workflow-<name>.prompt.md` → run `project-copilot.ps1`
4. **Adding a command**: Create `providers/claude-code/commands/<name>.md` → create `providers/github-copilot/prompts/<name>.prompt.md` → update CLI docs → run `project-copilot.ps1`
5. **Changing an agent or prompt**: Update the `.apm/` canonical first, then sync `providers/github-copilot/`, then run `project-copilot.ps1`
6. **Projecting to runtime**: `.apm/scripts/powershell/project-copilot.ps1` copies `providers/github-copilot/{agents,prompts,instructions}` → `.github/`
7. **Validation**: Run `python scripts/validate_copilot_assets.py` to check required files exist

## Canonical-only (no Copilot projection)

Some canonical assets are internal and do not need a Copilot runtime projection:

- `.apm/instructions/security-hardening.md` — applied internally by skills
- `.apm/instructions/canonical-layer.md` — for canonical layer governance
- `.apm/contexts/*.md` — consumed by workflows, not directly by Copilot
