This scenario can be used to create a new web application that is:

  • Deployable on the infrastructure of your choice,
  • composed of one or more backend and/or frontend components,
  • each component is versioned with a GitLab project on Innersource (public cloud),
  • each GitLab project will have its pre-configured CI/CD pipeline.

The first step is to enter some general information about your new application and pick a GitLab group on Innersource (public cloud) in which to create the new GitLab subgroup and/or project(s).

info

For applications with multiple components, we will create a GitLab project for each component. For applications with a single component, we will create a single GitLab project.

GitLab options

Please pick an existing GitLab group on Innersource (public cloud) in which to create the project(s). You can not use your personal namespace here.

For a multi-component application, it is advised to create a dedicated empty group.
