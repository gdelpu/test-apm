**Three layers**:

1. **Canonical** вҖ” `.apm/` defines all capabilities (source of truth)
2. **Adapter** вҖ” `providers/` document provider-specific conventions and mappings
3. **Runtime** вҖ” `.github/` (Copilot), `providers/claude-code/commands/` (Claude), `providers/cli/` (CLI)

## Source of Truth

- `.apm/` is the **canonical source** (Layer 1), including `.apm/knowledge/`.
- `providers/` holds adapter files вҖ” Copilot agents/prompts/instructions, Claude commands, CLI runner (Layer 2).
- `.github/` is the Copilot runtime projection, generated from `providers/github-copilot/` via `project-copilot.ps1` (Layer 3).
- Never edit `.github/` directly вҖ” always edit in `providers/github-copilot/` and regenerate.

## MCP Integration Layer

MCP (Model Context Protocol) servers provide optional tool enrichments that cross-cut all three layers:

```

в”Ӯ    14 curated servers, 4 profiles             в”Ӯ
в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”¬в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”¬в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ
```

- **Source of truth**: `.apm/contexts/mcp-registry.yaml` defines all servers, install configs, profiles, and fallbacks.
- **Projection**: `project-copilot.ps1` reads the registry and generates provider-specific MCP configuration files.
- **Skills**: Each MCP server has a dedicated skill in `.apm/skills/` plus enhancements to existing skills.
- **Fallback**: All MCP tools are optional вҖ” skills degrade gracefully when servers are unavailable.
- **Client overrides**: `clients/<name>/mcp-overrides.yaml` allows per-client MCP customisation.

> **Contributor guide**: [MCP Integration Guide](mcp-integration-guide.md) вҖ” how to add servers and enhance skills.
