## 7. CI Library integration

```yaml
variables:
  IAC_TOOL: terraform
  IAC_PATH: launchpad/
  IAC_ENV_PATH: launchpad/environments/$CI_ENVIRONMENT_NAME/variables.tfvars
```

---

## 8. Activation guide

1. Copy the `launchpad/` folder to the project repository root.
2. Fill in `environments/<env>/variables.tfvars` for each environment.
3. Configure the GitLab Terraform state backend in the CI/CD settings.
4. Trigger the `iac` deploy job from the DEP CI pipeline for the `dev` environment first.
5. Review the Terraform plan output in the pipeline before approving apply.
6. Promote to `staging`, then `prod` following the same pipeline promotion flow.

> **Security reminder:** never commit secrets or credentials in `variables.tfvars`. Use GitLab CI/CD variables or HashiCorp Vault (available via DEP Vault asset) for sensitive values.
