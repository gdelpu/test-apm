## 7. Activation guide

1. Copy the generated `.gitlab-ci.yml` to the project root.
2. Commit and push to a feature branch.
3. Open a Merge Request — the pipeline will trigger automatically.
4. Verify each stage passes in the GitLab CI/CD interface.
5. Merge to main to trigger the full pipeline including deployment.

> **First run tip:** disable jobs incrementally if the pipeline is too strict for project maturity. Use `ENABLE_*: "false"` / `DISABLE_*: "true"` CI Library variables.
