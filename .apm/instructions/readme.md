# README Maintenance Instructions

The root README.md is the canonical documentation for this repository.
It must reflect the actual state of the repo at all times.

## Self-maintenance rules

1. **When adding an agent**: update the Agents table (count + row).
2. **When adding a skill**: update the Skills table (count + row).
3. **When adding a workflow**: update the Workflows table (count + row).
4. **When adding a prompt**: update the Prompts table (count + row).
5. **When changing directory structure**: update the Repository Layout table.
6. **When changing provider projections**: update the Provider Setup section.

## Style

- Use Markdown tables for structured lists.
- Keep the architecture diagram current.
- Do not duplicate content from `.apm/` — reference it.
- Counts in section headers must match actual file counts.
